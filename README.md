# CS320

How can I ensure that my code, program, or software is functional and secure?
- I can ensure my code is functional and secure by writing unit tests, validating user input, and testing both normal and invalid scenarios. Thinking outside the box and not just testing the proper inputs, but also testing the invalid inputs will help validate all avenues that a user might take. Reviewing my code and fixing issues early also helps improve quality.

How do I interpret user needs and incorporate them into a program?
- I interpret user needs by carefully reviewing the project requirements and building features that meet those requirements. I also use testing to verify that the program behaves as the user expects.

How do I approach designing software?
- I approach software design by breaking the project into smaller parts, developing one feature at a time, and testing each part before moving on. This makes the software easier to build, debug, and maintain.
